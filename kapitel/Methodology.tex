This chapter describes the experimental setup used to analyse feature importance in neural network-based RUL prediction. It covers the dataset and preprocessing pipeline, the model architecture and its three configurations, the training procedure, and the implementation of feature importance methods. The models themselves are not the primary contribution of this thesis; they serve as controlled predictive systems under which feature importance behaviour can be systematically studied across different input configurations.

\section{Dataset and Data Preparation}

\subsection{Features set / sensors used}
The raw input space of each CMAPSS sub-dataset consists of three operational settings and 21 sensor measurements per time step. Eight sensors: s1, s5, s6, s10, s14, s16, s18, and s19, were identified as non-informative due to near-constant values across all trajectories and were excluded from the analysis. Each sensor channel is scaled independently to the range [0, 1] using a MinMax scaler. Because sensor readings vary systematically with operating condition, scaling is applied separately within each of the distinct operating regimes present in the data. The scaler is fitted exclusively on the training set.

Input sequences are constructed using a sliding window of length $T = 70$ time steps and stride 1, yielding tensors of shape (N x 13 x 70), where N is the number of sequences. The label associated with each window is the RUL value at the final time step of that window. The resulting input structure is illustrated in Figure \ref{fig:input_tensor}.

\subsection{Target definition (RUL label)}
The raw RUL for each engine trajectory is computed as the difference between the end-of-life cycle $t_{\text{EOL}}$ and the current cycle $t$:
\begin{equation}
    \text{RUL}_{\text{raw}}(t) = t_{\text{EOL}} - t
\end{equation}

A purely linear RUL label is known to dominate model training with the healthy regime, where no meaningful degradation signal is present. To address this, a piecewise linear transformation is applied using a maximum RUL threshold $R_c = \text{125}$ cycles. Early-life cycles, where the true RUL exceeds $R_c$, are assigned a constant label of $R_c$, under the assumption that the engine shows no detectable degradation in this regime. Only once the trajectory enters the degradation region does the label decrease linearly toward zero at failure:
\begin{equation}
    \text{RUL}(t) = min(\text{RUL}_{\text{raw}}(t), R_c).
\end{equation}

This transformation prevents the model from being overwhelmed by healthy-state samples and focuses learning on the degradation region, which carries the operationally relevant signal.

Following the piecewise transformation, RUL labels are normalized to [-1, 1] using a min-max scaling based on the training set:
\begin{equation}
    \hat{y} = 2 \cdot \frac{\text{RUL}(t) - \text{RUL}_{\text{min}}}{\text{RUL}_{\text{max}} - \text{RUL}_{\text{min}}} - 1,
\end{equation}

where $\text{RUL}_{\text{min}}$ and $\text{RUL}_{\text{max}} = R_c = \text{125}$ by construction, so that a healthy engine at the cap maps to +1 and an engine at failure maps to -1. The same scaling parameters are applied to the test set without re-fitting.

\section{Model Setup}
All model variants used in this thesis share a common architecture base on a Domain-Adversarial Neural Network (DANN) \cite{}. The DANN consists of three components: a convolutional feature extractor, a label predictor, and an optional domain classifier connected via a Gradient Reversal Layer (GRL). Since the primary purpose of the models here is to serve as a stable substrate for feature importance analysis rather than to maximise transfer performance, the domain classifier branch is used selectively and the architecture is held fixed across all configurations.

The \textbf{Feature Extractor} processes the input tensor of shape $(N x 13 x T)$ through four successive one-dimensional convolutional layers. The number of filters follows a halving schedule: 128, 64, 32, and 16 channels respectively, each with kernel size 5 and same-length padding of 2. After each convolutional layer, a LeakyReLU activation with negative slope 0.1 is applied. The temporal dimensions is collapsed using a global average pooling operation (AdaptiveAvgPool1d), which produces a fixed-length representation of 16 values regardless of input sequence length. A final fully connected layer projects this to a feature vector of length 64. Note that the slab height in Figure \ref{fig:dann_architecture} is a visual convention representing the reducing channel count and does not imply a reduction in the temporal dimension, which remains 70 throughout.

The \textbf{Label Predictor} maps the 64-dimensioinal feature vector to a scalar RUL estimate through two fully connected layers: Linear $(\text{64} \rightarrow \text{32})$ and Linear $(\text{32} \rightarrow \text{1})$, each preceded by a ReLU activation. The output is a single value representing the predicted RUL.

%The optional \textbf{Domain Classifier} receives the feature vector after it has passed through a Gradient Reversal lLayer, which negates gradients during backpropagation with a scaling factor $\alpha$. This adversarial signal encourages the featrure extractor to produce domain-invariant representations when training across multiples CMAPSS sub-datasets. The domain classifier architecture is a single fully connected layer with batch normalization. For the remainder of the thesis, the domain classifier is only used in the "All Models System" configuration, where data from all four CMAPSS sub-datasets are combined for training. In the single and pairwise model setups, the domain classifier is omitted to focus on feature importance within a single domain.

\section{Training Procedure}
All models are trained to minimise the mean squared error (MSE) between the predicted and true normalised RUL values:
\begin{equation}
    \mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}^{(i)} - y^{(i)})^2.
\end{equation}

The models are trained with the following subset combinations.

\subsection{Model Configurations}

Three training configurations are studied, differing only in the subset of CMAPSS data used during training. The four single-dataset models each train on one sub-dataset in isolation and serve as the baseline for feature importance behaviour under a single operating condition. The six pairwise models each train on the concatenated data from two sub-datasets, allowing examination of how feature importance shifts when a model must generalise across two distinct operating regimes simultaneously. The full-dataset model trains on all four sub-datasets combined and represents the most demanding generalisation scenario. \section{Implementation of Feature Importance Methods}
All eight feature importance methods are applied post-hoc to the trained models, without any modification of the learned weights. A central implementation challenge shared across all methods is the multivariate time-series structure of the input: each sample is a tensor of shape (N x 13 x 70), where the 13 sensor channels and 70 time steps together constitute the model's input spcae. Most standard feature importance implementations assume a tabular input where each column corresponds to an independent scalar feature. Applying these methods to a sequence model therefore requires deliberate desing dicisions about how features are defined, how perturbations are applied and how attributions over the temporal dimesnion are reduced to a per-sensor importance score. Unless stated otherwise, the full temporal extent of a sensor channel is treated as a single unit, perturbations are applied uniformly across all 70 time steps of a given sensor simultaneously and the resulting attributions are aggregated over time to yield a scalar importance score per sensor. This choice reflects the physically motivated view that a sensor's contribution to the RUL prediction should be attributed to the sensor as a whole rather than decomposed into individual time steps, which would result in a 910-dimensional attribution space that is difficult to interpret and compare across methods.

\textbf{Individual Conditional Expectation (ICE)}: For each of the 13 sensors, a uniform grid of 20 values is constructed between the 5th and 95th empirical percentile of that sensor's observed values across the dataset. The percentile bound are used in place of the global minimum and maximum to avoid evaluating the model in the tails of the distribution, where training data is sparse and prediction may be unreliable. For each grid value $g$ and each instance in the dataset, the entire time series of sensor $j$ is replaced by the constant value $g$, that is, $x_{j,t} = g$ for all $t \in [1, 70]$, while all remaining sensor channels are kept at their observed values. This produces one ICE curve per instance, tracing how the predicted RUL changes as sensor $j$ is swept across its range in isolation. The PDP is then computed as the pointwise mean of all ICE curves. Feature importance is derived as $\mathrm{ICE\text{-}FI}_j$ (see \ref{equation:ICE-FI}), measuring the mean variance of ICE curves across all instances.

\textbf{Feature Ablation}: The implementation follows the grouped ablation scheme provided by Captum's \textsl{FeatureAblation}, in which all 70 timesteps belonging to a given sensor channel are masked simultaneously as a single group rather than independently. This sensor-grouped formulation is essential for the present setting: ablating individual timesteps in isolation would produce attributions across 910 independent input positions, making cross-method comparison difficult and reducing the physical interpretability of the results. When sensor $j$ is ablated, its entire time series is replaced by the training set mean computed independently for each channel, representing the expected signal in the absence of informative variation. Attributions are defined as the absolute change in prediction relative to the unperturbed output, and global feature importance is obtained by averaging these absolute differences over all instances in the evaluation set.

\textbf{Shapley Additive Explanations}: To maintain consistency with the grouped ablation scheme described above, the 13 sensor channels are treated as 13 players in the cooperative game rather than operating on the 910-dimensional flattened input. Exact Shapley values are computed by enumerating all $2^{13} = \text{8192}$ possible sensor coalitions and evaluating the model on each, which is computationally tractable for this group size. When a sensor group is absent from a coalition, its timesteps are replaced by a mean valued baseline as in the ablation method. For each coalition $S$, the masked input is constructed by replacing the features of all sensors not in $S$ with the baseline and the model is queried to obtain $v(S)$. The Shapley values are then computed from these cached coalition outputs, yielding a per-sensor attribution for each instance in the dataset. A visualisation of the Shapley values is provided in the results chapter and implemented using the \textsl{shap} library.

\textbf{Local Interpretable Model-Agnostic Explanations}: The implementation uses Captum's \texttt{LIME} with a Lasso linear surrogate as the interpretable model class. Since LIME operates by perturbing the input in an interpretable feature space and fitting a weighted linear regression, the (13 x 70) input tensor is first flattened to a 910-dimensional vector. For each instance to be explained, 4000 perturbed samples are drawn in the neighbourhood of the input, each evaluated by the model. A proximity-weighted Lasso regression($\alpha = 0.01$) is then fitted to approximate the model locally, with the regularisation term enforcing sparsity in the resulting attribution vector. The zero vector is used as the reference baseline. After fitting, the 910-dimensional weight vector is reshaped back to (13 x 70) and the time dimension is averaged to obtain sensor-level importance scores. The Lasso regularisation parameter $\alpha$ was selected to balance sparsity with attribution stability across instances.

\textbf{Counterfactual Explanations}: Counterfactual explanations are generated using the \texttt{DiCE} library, which implements a gradient -based search for minimally modified inputs that shift the model's prediction into a specified target range. As \texttt{DiCE} requires a tabular input representation, the (13 x 70) tensor is flattened to a 910-dimensional feature vector before being passed to the \texttt{DiCE} data interface, which is initialised using the training set to define plausible feature ranges. For each query instance, the desired output range is specified as a window around the original prediction, so that the counterfactual represents a plausible alternative trajectory with a meaningfully different predicted RUL. The distance between the original and counterfactual input is penalised using an $\text{L}_1$ norm, which encourages sparse modifications, that is, changes concentrated in a small number of features. Feature importance is subsequently derived by computing the magnitude of change per sensor, aggregated across the 70 timesteps, which identifies which sensors were most frequently or substantially modified to achieve the target prediction shift.

\textbf{Permutation Feature Importance}: For each sensor channel, its values are permuted randomly across all N samples in the evaluation set, while all other channels remain at their original values. This destroys the statistical association between that sensor and the target RUL without altering the marginal distribution of the sensor's values, so any resulting drop in predictive performance can be attributed to the loss of that sensor's information. Performance is measured as MSE on the normalised RUL scale. To account for variability in the permutation, the procedure is repeated with 5 independent random permutations and the mean and standard deviation of the MSE increase are reported per sensor. The use of 5 repeats provides a sufficiently stable estimate for 13 sensors while keeping the total number of model evaluation manageable.

\textbf{Accumulated Local Effects}: The domain of each sensor is partitioned into 20 bins using quantile-based bin edges, so that each bin contains approximately the same number of samples. Quantile-based binning is preferred over uniform binning because sensor distributions may be skewed after operating-condition-specific normalisation and equal-frequency bins ensure that local effect estimates are based on sufficient data throughout the feature range. Bin membership for each sample is determined by the mean value of that sensor's time series, which reduces the (N x L) sensor values to a scalar per sample for assignment purposes. Within each bin, the local prediction difference is computed by replacing the entire time series of sensor $j$ with the lower and upper bin boundary values in turn, and recording the resulting change in model output. These local differences are averaged within each bin and then accumulated across bins following \ref{equation:ALE_center}. The accumulated effect curve is centered to zero mean over the empirical distribution of sensor values. A scalar importance score is derived as the range of the centered ALE curve, reflecting the total span of predicted RUL variation attributable to that sensor under the local conditional distribution.

\textbf{Integrated Gradients}: The implementation uses Captum's \texttt{IntegratedGradients} with 100 integration steps and the Gauss--Legendre quadrature approximation, which provides a more accurate estimate of the path integral than a simple Riemann sum for the same number of steps. The \texttt{multiply\_by\_inputs} flag is set to \texttt{True}, which scales each gradient by the difference between the input and the baseline. This ensures that the completeness property holds, i.e.\ the sum of attributions over all input dimensions equals the difference between the model's prediction at the input and at the baseline. A zero-valued tensor of shape $(1 \times 13 \times 70)$ is used as the baseline, consistent with the choice made for SHAP and Feature Ablation. Since all sensor values lie in $[0,1]$ after MinMax normalisation, a zero baseline represents a meaningful reference state corresponding to the minimum observed sensor value in each channel. Attributions are computed per instance over the full $(13 \times 70)$ input space, absolute values are taken to reflect attribution magnitude regardless of sign, and the temporal dimension is averaged to yield a per-sensor importance score of shape $(13,)$.

\section{Unified Feature Importance Representation}
Because the eight feature importance methods produce attributions in different units and on different scales, a unified representation is required before any cross method comparison can be made. For each method, the raw feature-level attributions are reduced to a single importance score per sensor by aggregating over the temporal dimension where applicable. The resulting scores are then normalised using $L_1$ normalisation:

\begin{equation}
    \text{FI}_{\text{L}_1} = \frac{|\text{FI}_j|}{\sum_{k=1}^{13} |\text{FI}_k|},
\end{equation}

so that the importance scores for all features sum to one for any given instance or aggregate. This normalisation ensures that the magnitude of attributions is comparable across methods regardless of their original scale, enabling direct visual and quantitative comparison in the results chapter.
